/pos-demo — Closing states (5 explicit sub-states)
Replaces current single "Closing" step
C·1 — Documents Preparing
Locked
Closing documents are being generated. No action needed — this typically takes 1–2 business days. User sees a clear "locked" state, not a spinner.
C·2 — Ready to Schedule Notary
Action needed
Documents ready. CTA unlocks: "Schedule your notary appointment." User picks time from available slots. Clear prompt — no ambiguity.
C·3 — Notary Scheduled
Waiting
Appointment confirmed. Shows date/time/notary name. "Add to calendar" secondary CTA. No further action — user is waiting for the appointment.
C·4 — Signing in Progress
Live
Notary session active. eNotary or in-person. Progress indicator shows documents being signed. "You're almost done" encouragement copy.
C·5 — Loan Closed ✓
Complete
All documents signed. Loan officially closed. Leads directly to FUNDED state. Celebratory moment — "Your solar installation is being scheduled with Westhaven."
These replace the current single CLOSING_PREP + ENOTARY_WAIT + SIGN_CLOSING states in POSDemo — which exist but have ambiguous UI. These 5 make each state's purpose and user action explicit.
